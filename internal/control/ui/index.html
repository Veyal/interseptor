CAPTURE POLICY

On long engagements, save database space by storing only in-scope traffic. Out-of-scope requests are still forwarded — they're just not written to history. Define your target under Target scope; with no scope set, everything is in scope (so this has no effect).

Toggle to keep only in-scope flows

BROWSER TELEMETRY

Chrome and Firefox send background traffic — Safe Browsing lookups, update pings, crash reports, captive-portal probes — that clutters history during web-app testing. This suppresses those hosts from capture and the intercept queue. Turn it off if you specifically need to inspect browser background traffic.

HOW TO TRUST IT
macOS
  1. Open the downloaded interceptor-ca.crt — Keychain Access opens.
  2. Add it to the System (or login) keychain.
  3. Double-click the Interceptor CA → expand Trust → set When using this certificate to Always Trust.
Windows
  1. Double-click the .crt → Install Certificate.
  2. Store location Current User → Place all certificates in the following store.
  3. Choose Trusted Root Certification Authorities → Next → Finish.
Firefox (uses its own store)
  1. Settings → Privacy & Security → Certificates → View Certificates.
  2. Authorities tab → Import → pick the .crt.
  3. Check Trust this CA to identify websites → OK.
iOS
  1. Open the .crt in Safari → Allow the profile download → install it in Settings.
  2. Settings → General → About → Certificate Trust Settings → enable full trust for the Interceptor CA.
Android
  1. Settings → Security → Encryption & credentials → Install a certificate → CA certificate.
  2. Note: user CAs are not trusted by most apps on Android 7+; testing app traffic may need a rooted device or a network-security-config.

TARGET SCOPE

With include rules, only matching traffic is in scope; exclude always wins; no rules = everything in scope. Scope focuses the ◎ in scope history toggle, the intercept gate, and the scanner. Tip: right-click a flow → Add to scope. Host supports a leading wildcard, e.g. *.acme.com; path is a prefix.

AI ASSIST · bring your own key

Optional. With your own provider key, the inspector's ✨ (or right-click → ✨ Ask AI) streams an explanation, suggests payloads you can load straight into Intruder, or summarizes findings. Off until a key is set; the exchange is sent to your chosen provider only when you ask.

SESSION · auth headers

Headers auto-applied to every Repeater and Intruder send (which bypass the proxy), so a token stays fresh without re-pasting. One Key: Value per line; these replace any existing value on the outgoing request. Typically an Authorization bearer token or a Cookie.

TOKEN MACRO (CSRF / RE-AUTH)

For apps that rotate a CSRF token every request or expire sessions: before each send, Interceptor fires a refresh request, pulls a value from its response with your regex, and injects it. Header mode sets a header; placeholder mode replaces a marker (e.g. §CSRF§) you put in the request/body.

Note: this fires one extra request per send — for Intruder that doubles the request count.
