SSL PINNING & MISSING TRAFFIC

            
Interseptor decrypts HTTPS only when the client trusts the CA. It does not bypass SSL pinning.

            
Checking traffic…

            
            
Verdicts
              
    
                
  • TLS blocked — the app sent CONNECT but rejected the MITM cert (pinning or untrusted CA). Look for red PIN rows in History.
    • 
                
  • No traffic — nothing reached the proxy. Check proxy settings, adb reverse, or whether the app ignores the system proxy.
    • 
                
  • No HTTPS — only cleartext HTTP captured, or HTTPS never went through the proxy.
    • 
                
  • OK — HTTPS interception is working.
    • 
              

            

            
Bypass pinning (on device)
              
    
                
  • Frida / objection — runtime hooks for OkHttp, custom TrustManagers, Flutter/Dart HTTP
    • 
                
  • Patched APK — repack with network_security_config or smali-patch pinning
    • 
                
  • Emulator + system CA — enough for apps that trust system CAs but don't pin
    • 
                
  • Root + Magisk / LSPosed — persistent bypass modules
    • 
              

            

          

          

            

              
HOW TO TRUST IT

              
macOS
    
                
  1. Open the downloaded interseptor-ca.crt — Keychain Access opens.
    2. 
                
  2. Add it to the System (or login) keychain.
    3. 
                
  3. Double-click the Interseptor CA → expand Trust → set When using this certificate to Always Trust.
    4. 
              

              
Windows
    
                
  1. Double-click the .crt → Install Certificate.
    2. 
                
  2. Store location Current User → Place all certificates in the following store.
    3. 
                
  3. Choose Trusted Root Certification Authorities → Next → Finish.
    4. 
              

              
Firefox (uses its own store)
    
                
  1. Settings → Privacy & Security → Certificates → View Certificates.
    2. 
                
  2. Authorities tab → Import → pick the .crt.
    3. 
                
  3. Check Trust this CA to identify websites → OK.
    4. 
              

              
iOS
    
                
  1. Use the Mobile devices tab (iOS) — Setup all for Simulator, or download the profile for a physical iPhone.
    2. 
                
  2. Settings → General → VPN & Device Management → install the Interseptor profile.
    3. 
                
  3. Settings → General → About → Certificate Trust Settings → enable full trust for the Interseptor CA.
    4. 
              

              
Android
    
                
  1. If adb is installed, use the Mobile devices tab (Android) for one-click proxy and CA install.
    2. 
                
  2. Manual: Settings → Security → Encryption & credentials → Install a certificate → CA certificate.
    3. 
                
  3. Note: user CAs are not trusted by most apps on Android 7+; testing app traffic may need a rooted device or a network-security-config.
    4. 
              

              
Linux (system / CLI)
    
                
  1. Debian/Ubuntu: copy to /usr/local/share/ca-certificates/interseptor.crt → run sudo update-ca-certificates.
    2. 
                
  2. Fedora/RHEL: copy to /etc/pki/ca-trust/source/anchors/ → run sudo update-ca-trust.
    3. 
                
  3. curl/wget one-off: curl --cacert ~/.interseptor/ca/ca.crt -x http://127.0.0.1:8080 https://…. Many tools also honour SSL_CERT_FILE/REQUESTS_CA_BUNDLE pointing at the CA.
    4.
